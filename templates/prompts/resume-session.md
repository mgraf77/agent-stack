# Prompt: Resume a stopped session

Use this instead of re-describing a task from scratch after an
interruption, a new session, or a tool switch.

```
Resume work on GitHub issue #<NUMBER>, branch <BRANCH NAME>
(PR #<NUMBER IF ONE EXISTS>).

Before doing anything:
1. Read the issue for the original scope and definition of done.
2. Read the current diff/commits on the branch and any PR review
   comments or CI output to see what's already done vs. still open.
3. Tell me your read of what's left before you start.

Then continue from there — don't redo work that's already committed, and
don't expand scope beyond the original issue unless I say so.
```
