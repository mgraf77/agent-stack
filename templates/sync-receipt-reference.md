# Reference: `sync-receipt.json`

This is not a fill-in-the-blanks template — `sync-receipt.json` is
generated by `node scripts/sync.mjs --mode apply` and written into every
adapter's target directory (`.claude/skills/sync-receipt.json`,
`.agents/skills/sync-receipt.json`) in the product repo. Never hand-write
or hand-edit one; `node scripts/doctor.mjs` will flag a hand-edited receipt
as drift. This doc exists so you can read one without guessing at the
fields. A filled example is in `examples/sync-receipt.example.json`.

```json
{
  "receiptVersion": "1",
  "profile": "<profile_id from profiles/<profile_id>.json>",
  "sourceRelease": "<the --release value passed to sync.mjs, e.g. a tag or commit>",
  "adapter": { "id": "<adapter id, e.g. claude-code or codex>", "targetDir": "<e.g. .claude/skills>" },
  "generatedAt": "<ISO-8601 timestamp of the apply run>",
  "skills": [
    {
      "id": "<skill id>",
      "files": [{ "path": "<file path relative to the skill dir>", "sha256": "<checksum>" }],
      "skillChecksum": "<checksum over the skill's files>"
    }
  ],
  "receiptChecksum": "<checksum over the whole receipt, recomputed by doctor.mjs to detect drift>"
}
```

Fields worth knowing when reading a receipt or debugging `doctor.mjs`
output:

- `sourceRelease` is only as good as what was passed to `--release` at
  apply time. Until Agent Stack has cut real tagged releases, this may
  read `"unreleased"` — see `release/CHECKLIST.md`.
- `skills[].files[].sha256` and `skillChecksum` are what `doctor.mjs`
  recomputes from what's actually on disk; any mismatch means the synced
  files drifted from what was pinned.
- `receiptChecksum` covers the receipt itself, so a hand-edited receipt
  (not just hand-edited skill files) is also caught.
